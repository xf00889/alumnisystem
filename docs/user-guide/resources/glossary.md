# Glossary

This glossary defines key terms used throughout the NORSU Alumni System documentation.

## General Terms

**Alumni**
: A graduate or former student of NORSU (Negros Oriental State University).

**System**
: The NORSU Alumni System web application.

**End User**
: Any person who interacts with the System, including regular alumni users and administrators.

**Regular User**
: An authenticated alumni member with standard access privileges.

**Admin User**
: A staff member with elevated privileges to manage System content and users.

**Feature**
: A distinct functional capability within the System.

**Dashboard**
: The main overview page showing key information and quick access to features.

## Account & Authentication

**Authentication**
: The process of verifying a user's identity through login credentials.

**Registration**
: The process of creating a new user account in the System.

**Email Verification**
: The process of confirming a user's email address by clicking a link sent via email.

**Session**
: A period of authenticated access to the System after logging in.

**Password Reset**
: The process of creating a new password when the current password is forgotten.

**Two-Factor Authentication (2FA)**
: An additional security layer requiring a second form of verification beyond password.

## Profile & Directory

**Profile**
: A user's personal page containing information about their education, work experience, and skills.

**Alumni Directory**
: A searchable database of all registered alumni users.

**Education History**
: Records of academic institutions attended, degrees earned, and graduation dates.

**Work Experience**
: Records of employment history including job titles, companies, and dates.

**Skills**
: Professional competencies and abilities listed on a user profile.

**Privacy Settings**
: Controls that determine what information is visible to other users.

## Networking & Communication

**Connection**
: A mutual relationship between two users allowing enhanced interaction.

**Connection Request**
: An invitation sent to another user to establish a connection.

**Direct Message (DM)**
: A private one-on-one conversation between connected users.

**Group Chat**
: A conversation involving multiple participants.

**Notification**
: An alert about system activity relevant to the user.

## Events

**Event**
: A scheduled gathering or activity for alumni.

**RSVP**
: A response indicating whether a user plans to attend an event (from French "répondez s'il vous plaît").

**Public Event**
: An event visible to all visitors, including non-registered users.

**Private Event**
: An event visible only to registered users or specific groups.

**Event Organizer**
: The user or admin who created and manages an event.

## Jobs & Career

**Job Posting**
: An advertisement for an employment opportunity.

**Internal Job**
: A position posted by the university or alumni association requiring application through the System.

**External Job**
: A position posted from an external source with a link to apply elsewhere.

**Job Application**
: A submission of interest and qualifications for a job posting.

**Applicant**
: A user who has applied for a job posting.

**Skill Matching**
: The process of recommending jobs based on a user's listed skills.

## Mentorship

**Mentor**
: An experienced alumni member who provides guidance to others.

**Mentee**
: A user seeking guidance from a mentor.

**Mentorship Request**
: An invitation to establish a mentor-mentee relationship.

**Mentorship Session**
: A scheduled meeting between mentor and mentee.

## Donations & Campaigns

**Campaign**
: A fundraising initiative for a specific cause or project.

**Donation**
: A monetary contribution to a campaign.

**Payment Proof**
: Documentation (receipt or screenshot) verifying a donation payment.

**GCash**
: A mobile wallet payment service used in the Philippines.

**Campaign Goal**
: The target amount a campaign aims to raise.

**Campaign Status**
: The current state of a campaign (active, completed, cancelled).

## Groups

**Alumni Group**
: A community of users organized around a common interest, graduation year, or location.

**Group Member**
: A user who has joined a group.

**Group Admin**
: A user with permissions to manage group settings and membership.

**Group Discussion**
: A conversation thread within a group.

**Security Question**
: A question that must be answered correctly to join certain groups.

## Surveys & Feedback

**Survey**
: A questionnaire designed to collect information from users.

**Survey Response**
: A completed submission of a survey.

**Question Type**
: The format of a survey question (multiple choice, text, rating, etc.).

**Feedback**
: User-submitted comments, suggestions, or issue reports about the System.

**Feedback Category**
: A classification of feedback type (bug report, feature request, general comment).

## Content Management

**CMS (Content Management System)**
: The administrative interface for managing website content.

**Page Section**
: A content block on a webpage (hero, about, features, etc.).

**Timeline Item**
: An entry in the historical timeline of the institution.

**Testimonial**
: A user statement or review displayed on the website.

**FAQ (Frequently Asked Questions)**
: A list of common questions and their answers.

## Administration

**User Management**
: Administrative tools for viewing and managing user accounts.

**User Role**
: A classification determining a user's permissions (regular user, admin, staff).

**Analytics**
: Statistical data about system usage and user activity.

**Report**
: A generated document containing system data or statistics.

**Export**
: The process of downloading data in a file format (CSV, Excel, PDF).

**Bulk Action**
: An operation performed on multiple items simultaneously.

## Technical Terms

**Filter**
: A tool to narrow down search results based on specific criteria.

**Search**
: The process of finding specific content using keywords.

**Pagination**
: Dividing content across multiple pages for easier navigation.

**Upload**
: The process of transferring a file from a local device to the System.

**Download**
: The process of transferring a file from the System to a local device.

**Attachment**
: A file included with a message, feedback, or application.

**Screenshot**
: An image capture of what appears on a screen.

**URL (Uniform Resource Locator)**
: A web address for accessing a specific page or resource.

**Browser**
: Software used to access websites (Chrome, Firefox, Safari, Edge).

**Cache**
: Temporary storage of web data to improve loading speed.

## Status Indicators

**Active**
: Currently operational or available.

**Inactive**
: Not currently operational or available.

**Pending**
: Awaiting action or approval.

**Approved**
: Accepted or authorized.

**Rejected**
: Declined or not approved.

**Draft**
: A preliminary version not yet published.

**Published**
: Made available for viewing by intended audience.

**Archived**
: Stored for historical reference but no longer active.

---

**Need Help?** If you encounter a term not defined here, please contact system administrators.
