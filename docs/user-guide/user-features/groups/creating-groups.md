# Creating Groups

Learn how to create new alumni groups, configure group settings, and set up security questions for member verification.

## Overview

Creating an alumni group allows you to bring together alumni with shared interests, batch years, or professional goals. As a group creator, you become the group admin with full control over settings, membership, and content moderation.

## Who Can Use This Feature

- **Required**: Authenticated alumni account
- **Access Level**: All registered alumni users
- **Prerequisites**: 
  - Verified email address
  - Completed profile setup
  - Good standing in the system

## How to Access

1. Log in to your alumni account
2. Navigate to **"Groups"** in the main navigation
3. Click the **"Create Group"** button (top right of the page)
4. You will be directed to the Group Creation form

**Alternative Access:**
- From your dashboard, click **"Create New Group"**
- From any group page, click **"Create Group"** in the header

## Before Creating a Group

### Check for Existing Groups

Before creating a new group, search for existing groups to avoid duplicates:

**Step-by-Step:**

1. Use the search function on the Groups List page
2. Search for keywords related to your intended group
3. Filter by batch year, course, and campus
4. Review existing groups carefully
5. Only create a new group if none exists

**Why This Matters:**
- Prevents duplicate groups
- Consolidates alumni in one place
- Increases group engagement
- Reduces confusion for members

### Plan Your Group

Consider these questions before creating:

**Purpose:**
- What is the main purpose of this group?
- Who is the target audience?
- What activities will the group focus on?

**Membership:**
- Who should be able to join?
- Should membership require approval?
- Do you need security questions?

**Content:**
- What type of content will be shared?
- Should posts require approval?
- How will you moderate content?

**Management:**
- Who will help manage the group?
- How often will you check the group?
- What are your group rules?

## Group Creation Process

The system uses a two-phase creation process to ensure groups are properly configured and prevent duplicates.

### Phase 1: Basic Criteria

This phase collects information to check for duplicate groups.

**Step-by-Step:**

1. Click **"Create Group"** button
2. You are directed to Phase 1 form
3. Fill in the required fields:
   - **Batch Start Year**: Beginning year of the batch (e.g., 2015)
   - **Batch End Year**: Ending year of the batch (e.g., 2019)
   - **Course**: Select your course from the dropdown
   - **Campus**: Select the campus from the dropdown
4. Click **"Next"** to proceed to Phase 2

**Field Details:**

**Batch Start Year:**
- Required field
- Must be between 1900 and current year
- Represents when the batch started

**Batch End Year:**
- Required field
- Must be between 1900 and current year
- Must be equal to or greater than start year
- Represents when the batch graduated

**Course:**
- Required field
- Select from available courses
- Examples: BS Computer Science, BS Engineering, etc.

**Campus:**
- Required field
- Select from NORSU campuses:
  - NORSU Main Campus
  - NORSU Guihulngan Campus
  - NORSU Siaton Campus
  - NORSU Pamplona Campus
  - NORSU Bais Campus
  - NORSU BSC (Bayawan-Sta. Catalina) Campus

**Expected Result:**
- System checks for duplicate groups
- If duplicate exists, error message appears with existing group name
- If no duplicate, you proceed to Phase 2

**Duplicate Detection:**

If a group with the same criteria exists, you'll see:
- Error message: "A group with the same batch years (YYYY-YYYY), course (Course Name), and campus (Campus Name) already exists."
- Link to the existing group
- Suggestion to join the existing group instead

### Phase 2: Group Details and Settings

This phase collects detailed information about your group.

**Step-by-Step:**

1. After Phase 1 validation, you're directed to Phase 2
2. Review the Phase 1 data displayed at the top
3. Fill in the group details:

**Group Name** (Required):
- Enter a descriptive name for your group
- Example: "BSCS 2015-2019 Main Campus"
- Keep it clear and searchable
- Maximum 255 characters

**Group Type** (Required):
- Select one:
  - **Automatic**: For batch-based groups
  - **Manual**: For interest or special purpose groups
  - **Hybrid**: Combination of both
- Choose based on your group's purpose

**Description** (Required):
- Provide a detailed description
- Explain the group's purpose
- Mention target audience
- Include any rules or expectations
- Use clear, welcoming language

**Visibility** (Required):
- Select one:
  - **Public**: Visible to all users, anyone can request to join
  - **Private**: Only visible to members, invitation-only
  - **Restricted**: Visible to all, but requires security questions
- Choose based on your privacy needs

**Tags** (Optional):
- Add relevant tags separated by commas
- Examples: "computer science, technology, programming"
- Helps with search and discovery
- Maximum 10 tags recommended

**Cover Image** (Optional):
- Upload a cover image for your group
- Recommended size: 1200x400 pixels
- Accepted formats: JPG, PNG, GIF
- Maximum file size: 5MB
- Represents your group visually

**Profile Photo** (Optional):
- Upload a profile photo/logo
- Recommended size: 400x400 pixels
- Accepted formats: JPG, PNG, GIF
- Maximum file size: 2MB
- Appears in group listings

4. Configure group settings:

**Requires Approval** (Checkbox):
- Check if join requests need admin approval
- Unchecked: Members join instantly
- Checked: Admins must approve each request
- Recommended for controlled membership

**Has Security Questions** (Checkbox):
- Check if members must answer security questions
- Used for verification purposes
- Questions are set up after group creation
- Recommended for batch-specific groups

**Require Post Approval** (Checkbox):
- Check if posts need approval before publishing
- Helps maintain content quality
- Admins/moderators review posts
- Recommended for large or public groups

**Maximum Members** (Optional):
- Set a limit on group size
- Leave blank for unlimited
- Useful for exclusive or small groups
- Must be at least 1 if specified

5. Review all information carefully
6. Click **"Create Group"** button

**Expected Result:**
- Group is created successfully
- You are automatically added as admin
- You are redirected to the group detail page
- Success message appears
- Group appears in your "My Groups" list

## Configuring Group Settings

### Setting Up Security Questions

If you enabled security questions during creation:

**Step-by-Step:**

1. Navigate to your group detail page
2. Click **"Manage Members"** (admin only)
3. Click **"Security Questions"** tab
4. Click **"Add Question"** button
5. Enter your question in the text field
6. Check **"Required"** if the question must be answered
7. Click **"Save Question"**
8. Repeat for additional questions (recommended: 3-5 questions)

**Question Examples:**

For Batch Groups:
- "Who was your class adviser?"
- "Name three classmates from your section"
- "What was your thesis/capstone project title?"
- "Name a memorable event from your batch"
- "Who was the dean during your time?"

For Campus Groups:
- "Name a building on campus"
- "What was the campus mascot?"
- "Name a popular hangout spot on campus"

For Course Groups:
- "Name a professor in your department"
- "What was your favorite subject?"
- "Name a lab or facility in your department"

**Best Practices:**
- Ask questions only your target members can answer
- Avoid questions that are too easy or too hard
- Make at least 2-3 questions required
- Update questions periodically
- Don't ask for sensitive personal information

**Expected Result:**
- Questions are saved and active
- New members must answer when joining
- You can review answers before approving members
- Questions can be edited or deleted later

### Managing Group Settings

After creation, you can update settings:

**Step-by-Step:**

1. Go to your group detail page
2. Click **"Edit Group"** button (admin only)
3. Update any field except:
   - Batch years (locked after creation)
   - Course (locked after creation)
   - Campus (locked after creation)
4. Click **"Save Changes"**

**Editable Settings:**
- Group name
- Description
- Visibility
- Tags
- Cover image
- Profile photo
- Requires approval
- Has security questions
- Require post approval
- Maximum members

**Expected Result:**
- Changes are saved immediately
- Members see updated information
- Activity is logged
- Notification may be sent to members

## Group Types Explained

### Automatic Groups

**Purpose:** Batch-based groups for specific graduation years

**Characteristics:**
- Based on batch year, course, and campus
- System prevents duplicates
- Intended for classmates and batchmates
- Usually requires security questions

**When to Use:**
- Creating a group for your graduation year
- Organizing batch reunions
- Connecting with former classmates
- Batch-specific activities

**Example:**
- "BS Computer Science 2015-2019 Main Campus"
- "BS Engineering 2010-2014 Guihulngan Campus"

### Manual Groups

**Purpose:** Interest-based or special purpose groups

**Characteristics:**
- Not tied to specific batch years
- Can be for any purpose or interest
- More flexible membership criteria
- May or may not require approval

**When to Use:**
- Professional networking groups
- Hobby or interest groups
- Alumni associations
- Special projects or initiatives

**Examples:**
- "NORSU IT Professionals Network"
- "NORSU Basketball Alumni"
- "NORSU Entrepreneurs Club"
- "NORSU Environmental Advocates"

### Hybrid Groups

**Purpose:** Combination of automatic and manual features

**Characteristics:**
- Has batch criteria but open to others
- Flexible membership rules
- Can include multiple batches
- Balanced approach

**When to Use:**
- Multi-batch groups (e.g., 2015-2020)
- Department-wide groups
- Campus-wide initiatives
- Cross-batch collaborations

**Examples:**
- "NORSU CS Department Alumni (2010-2020)"
- "NORSU Main Campus Engineering Alumni"
- "NORSU Research Collaborators"

## Visibility Options Explained

### Public Groups

**Characteristics:**
- Visible to all users in the system
- Appears in search results
- Group information is public
- Anyone can request to join

**Best For:**
- Large batch groups
- Open networking groups
- Campus-wide groups
- Public interest groups

**Privacy Level:** Low - Maximum visibility

### Private Groups

**Characteristics:**
- Only visible to current members
- Does not appear in public searches
- Invitation-only membership
- Group content is hidden

**Best For:**
- Exclusive groups
- Small, close-knit groups
- Sensitive discussions
- Executive or leadership groups

**Privacy Level:** High - Maximum privacy

### Restricted Groups

**Characteristics:**
- Visible to all users
- Requires security questions to join
- Group info is public, content is private
- Verification before membership

**Best For:**
- Batch verification groups
- Alumni verification groups
- Controlled membership with transparency
- Groups requiring identity verification

**Privacy Level:** Medium - Balanced approach

## Tips and Best Practices

### Planning Your Group

**Do:**
- Research existing groups thoroughly
- Define a clear purpose
- Plan your moderation strategy
- Set realistic expectations
- Prepare welcome messages

**Don't:**
- Create duplicate groups
- Make overly broad groups
- Neglect group management
- Set unrealistic rules
- Ignore member feedback

### Naming Your Group

**Good Names:**
- Clear and descriptive
- Include key identifiers (batch, course, campus)
- Easy to search for
- Professional and appropriate

**Examples:**
- "BSIT 2018-2022 Main Campus"
- "NORSU Engineering Alumni Network"
- "NORSU Dumaguete Professionals"

**Avoid:**
- Vague names ("The Group")
- Inside jokes or slang
- Overly long names
- Special characters or emojis

### Writing Descriptions

**Include:**
- Group purpose and goals
- Target audience
- Expected activities
- Group rules and guidelines
- Contact information

**Structure:**
```
Welcome to [Group Name]!

This group is for [target audience] to [main purpose].

What we do:
- [Activity 1]
- [Activity 2]
- [Activity 3]

Group Rules:
1. [Rule 1]
2. [Rule 2]
3. [Rule 3]

Join us to [benefit/call to action]!
```

### Setting Up Security Questions

**Effective Questions:**
- Specific to your target audience
- Not easily guessable
- Verifiable by admins
- Respectful and appropriate
- Mix of easy and moderate difficulty

**Question Balance:**
- 3-5 questions total
- 2-3 required questions
- 1-2 optional questions
- Mix of factual and experiential

### Managing Your New Group

**First Week:**
- Welcome new members
- Post introduction and guidelines
- Encourage member introductions
- Set the tone for the group
- Be active and responsive

**Ongoing:**
- Post regularly
- Respond to member questions
- Moderate content appropriately
- Organize events or activities
- Gather member feedback

## Common Use Cases

### Creating a Batch Group

**Scenario:** You want to create a group for your graduation year

**Steps:**
1. Verify no existing group for your batch
2. Click "Create Group"
3. Enter your batch years, course, and campus
4. Proceed to Phase 2
5. Name it clearly (e.g., "BSCS 2015-2019 Main Campus")
6. Choose "Automatic" type
7. Set visibility to "Restricted"
8. Enable "Requires Approval"
9. Enable "Has Security Questions"
10. Create the group
11. Set up 3-5 security questions
12. Invite your batchmates

### Creating an Interest Group

**Scenario:** You want to create a professional networking group

**Steps:**
1. Click "Create Group"
2. Enter broad batch years or current year
3. Select a general course or your field
4. Choose your campus or main campus
5. Proceed to Phase 2
6. Name it descriptively (e.g., "NORSU IT Professionals")
7. Choose "Manual" type
8. Set visibility to "Public"
9. Enable "Requires Approval" for quality control
10. Disable security questions
11. Write a detailed description
12. Add relevant tags
13. Create the group
14. Start inviting members

### Creating a Campus-Wide Group

**Scenario:** You want to create a group for all alumni from your campus

**Steps:**
1. Click "Create Group"
2. Enter a wide batch year range
3. Select a general course or "All Courses"
4. Choose your specific campus
5. Proceed to Phase 2
6. Name it (e.g., "NORSU Guihulngan Campus Alumni")
7. Choose "Hybrid" type
8. Set visibility to "Public"
9. Enable "Requires Approval"
10. Consider security questions
11. Write an inclusive description
12. Add campus-related tags
13. Create the group
14. Promote to campus alumni

## Troubleshooting

### Cannot Create Group - Duplicate Exists

**Problem:** Error message about existing group

**Solutions:**
- Review the existing group mentioned
- Join the existing group instead
- Contact existing group admin about merging
- Adjust your criteria if truly different
- Create a more specific group if needed

### Form Validation Errors

**Problem:** Cannot submit the form

**Solutions:**
- Check all required fields are filled
- Verify batch end year â‰¥ start year
- Ensure name is not too long
- Check file sizes for images
- Review error messages carefully
- Fix highlighted fields

### Images Not Uploading

**Problem:** Cover image or profile photo won't upload

**Solutions:**
- Check file size (max 5MB for cover, 2MB for profile)
- Verify file format (JPG, PNG, GIF only)
- Try a different image
- Compress the image
- Check internet connection
- Try a different browser

### Security Questions Not Saving

**Problem:** Questions don't save or disappear

**Solutions:**
- Ensure "Has Security Questions" is enabled
- Check question text is not empty
- Verify you have admin permissions
- Refresh the page and try again
- Contact support if issue persists

### Group Not Appearing in List

**Problem:** Created group doesn't show up

**Solutions:**
- Refresh the Groups List page
- Check "My Groups" section
- Verify group was actually created
- Check if group is set to Private
- Clear browser cache
- Contact support if needed
