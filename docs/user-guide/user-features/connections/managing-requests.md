# Managing Requests

This guide explains how to view, accept, and reject incoming and outgoing connection requests.

## Overview

The Connection Requests page is your central hub for managing all connection activity. Here you can review incoming requests from alumni who want to connect with you, and track outgoing requests you've sent to others.

## Who Can Manage Connection Requests

- **Required**: Authenticated alumni account
- **Access Level**: All registered alumni users
- **Prerequisites**: 
  - Completed profile setup
  - Verified email address

## Accessing Connection Requests

### Step 1: Navigate to Connection Requests

**Method 1: Main Navigation**
1. Log in to your NORSU Alumni account
2. Click on **"Connections"** in the main navigation menu
3. Select **"Connection Requests"** or **"Requests"** from the dropdown

**Method 2: Notification Badge**
1. Look for a notification badge on the Connections menu
2. The badge shows the number of pending incoming requests
3. Click on the badge or menu item to view requests

**Method 3: Direct Link**
1. Some notifications include direct links to connection requests
2. Click the link in the notification or email
3. You'll be taken directly to the requests page

**Expected Result**: The Connection Requests page loads, showing two tabs:
- **Incoming Requests**: Requests sent to you
- **Outgoing Requests**: Requests you've sent

![Connection Requests Page](../../screenshots/connection-requests-page.png)

## Understanding the Requests Page

### Page Layout

The Connection Requests page is divided into two main sections:

1. **Incoming Requests Tab**
   - Shows requests from alumni who want to connect with you
   - Displays accept and reject buttons
   - Shows requester's profile information
   - Badge indicates number of pending requests

2. **Outgoing Requests Tab**
   - Shows requests you've sent to other alumni
   - Displays pending status
   - Shows recipient's profile information
   - May include cancel option

### Request Card Information

Each request card displays:
- **Profile Photo**: The user's profile picture or default avatar
- **Full Name**: First and last name of the user
- **Username**: System username
- **Current Position**: Job title (if provided)
- **Graduation Info**: Year and course
- **Location**: City and province (if provided)
- **Request Date**: When the request was sent
- **Action Buttons**: Accept/Reject or status indicators

![Request Card Example](../../screenshots/request-card.png)

## Managing Incoming Requests

Incoming requests are from alumni who want to connect with you.

### Viewing Incoming Requests

#### Step 1: Access Incoming Requests Tab

1. Navigate to the Connection Requests page
2. Click on the **"Incoming Requests"** tab (usually selected by default)
3. View the list of pending requests

**Expected Result**: You'll see all pending connection requests sent to you, with the most recent at the top.

![Incoming Requests Tab](../../screenshots/incoming-requests.png)

#### Step 2: Review Request Details

For each request, review:
- **Who is requesting**: Name and profile photo
- **Their background**: Current position, graduation year, course
- **Mutual connections**: Shared connections (if displayed)
- **Profile completeness**: How complete their profile is

**Tip**: Click on the user's name or profile photo to view their full profile before deciding.

### Accepting Connection Requests

#### Step 1: Review the Request

1. Read the requester's information on the request card
2. Optionally, click their name to view their full profile
3. Decide if you want to connect with this person

#### Step 2: Click Accept

1. Locate the **"Accept"** button on the request card (usually green)
2. Click the **"Accept"** button
3. The system will process your acceptance

**Expected Result**:
- A success message appears: "You are now connected with [Alumni Name]"
- The request is removed from your incoming requests list
- The user is added to your connections list
- The requester receives a notification
- You can now message each other

![Accept Connection Request](../../screenshots/accept-request.png)

#### Step 3: Verify Connection

1. Navigate to **"My Connections"** to verify
2. The newly connected alumni should appear in your connections list
3. You can now message them or view their full profile

### Rejecting Connection Requests

#### Step 1: Review the Request

1. Read the requester's information
2. Decide if you want to reject the request

**Note**: Rejecting a request is permanent. The requester may not be able to send another request immediately.

#### Step 2: Click Reject

1. Locate the **"Reject"** button on the request card (usually red or gray)
2. Click the **"Reject"** button
3. The system will process your rejection

**Expected Result**:
- A confirmation message appears: "Connection request rejected"
- The request is removed from your incoming requests list
- The requester may receive a notification (depending on settings)
- The request status changes to "Rejected"

![Reject Connection Request](../../screenshots/reject-request.png)

#### Step 3: Confirmation

The request is immediately removed from your incoming requests list. There is no undo option, but the user may be able to send another request in the future (depending on system settings).

### Bulk Actions (If Available)

Some systems allow bulk actions on requests:

1. **Select Multiple Requests**: Check boxes next to multiple requests
2. **Choose Action**: Select "Accept All" or "Reject All"
3. **Confirm**: Confirm the bulk action
4. **Process**: All selected requests are processed at once

**Note**: This feature may not be available in all versions.

## Managing Outgoing Requests

Outgoing requests are connection requests you've sent to other alumni.

### Viewing Outgoing Requests

#### Step 1: Access Outgoing Requests Tab

1. Navigate to the Connection Requests page
2. Click on the **"Outgoing Requests"** tab
3. View the list of pending requests you've sent

**Expected Result**: You'll see all connection requests you've sent that are still pending, with the most recent at the top.

![Outgoing Requests Tab](../../screenshots/outgoing-requests.png)

#### Step 2: Review Request Status

For each outgoing request, you can see:
- **Recipient**: Who you sent the request to
- **Date Sent**: When you sent the request
- **Status**: "Pending" (waiting for response)
- **Profile Info**: Recipient's current position and details

### Understanding Outgoing Request Status

Outgoing requests can have several states:

1. **Pending**: 
   - Request sent but not yet responded to
   - Waiting for recipient to accept or reject
   - No action needed from you

2. **Accepted**: 
   - Request was accepted
   - You're now connected
   - Request moves to connections list

3. **Rejected**: 
   - Request was declined
   - May be removed from list
   - You may not be able to send another immediately

### Canceling Outgoing Requests (If Available)

Some systems allow you to cancel pending requests:

#### Step 1: Locate the Request

1. Find the request you want to cancel in the outgoing requests list
2. Look for a **"Cancel"** or **"Withdraw"** button

#### Step 2: Cancel the Request

1. Click the **"Cancel"** or **"Withdraw"** button
2. Confirm the cancellation if prompted
3. The system will process the cancellation

**Expected Result**:
- The request is removed from your outgoing requests list
- The recipient no longer sees your request
- You can send a new request in the future if desired

**Note**: This feature may not be available in all versions of the system.

## Notifications

### Incoming Request Notifications

When someone sends you a connection request:
- **In-App Notification**: Badge appears on Connections menu
- **Email Notification**: You may receive an email (depending on settings)
- **Notification Center**: Alert appears in your notification center
- **Badge Count**: Number shows how many pending requests you have

### Accepted Request Notifications

When someone accepts your connection request:
- **In-App Notification**: You receive a notification
- **Email Notification**: You may receive an email (depending on settings)
- **Success Message**: Confirmation that you're now connected

### Managing Notification Settings

To adjust notification preferences:
1. Go to **"Account Settings"** or **"Profile Settings"**
2. Navigate to **"Notifications"** section
3. Toggle connection request notifications on/off
4. Choose email notification preferences
5. Save your settings

## Best Practices

### Responding to Incoming Requests

1. **Review Promptly**: Check requests regularly and respond in a timely manner
2. **View Profiles**: Click through to view full profiles before deciding
3. **Be Selective**: Only accept requests from people you want in your network
4. **Consider Context**: Look for mutual connections or shared interests
5. **Professional Standards**: Maintain professional networking standards

### Managing Outgoing Requests

1. **Be Patient**: Give recipients time to respond
2. **Don't Spam**: Avoid sending too many requests at once
3. **Quality Over Quantity**: Focus on meaningful connections
4. **Follow Up**: Once accepted, send a message to introduce yourself
5. **Review Regularly**: Check your outgoing requests periodically

### Decision Criteria

**Consider Accepting If**:
- You know the person from school or work
- You have mutual connections
- They're in your field or industry
- Their profile is complete and professional
- You share common interests or goals

**Consider Rejecting If**:
- You don't know the person and have no mutual connections
- Their profile is incomplete or unprofessional
- You have no common interests or professional overlap
- The request seems suspicious or spam-like

## Troubleshooting

### Cannot See Incoming Requests

**Problem**: Incoming requests tab is empty but you expect requests

**Solutions**:
1. Refresh the page
2. Check if notifications indicate pending requests
3. Verify your internet connection
4. Log out and log back in
5. Clear browser cache
6. Check if requests were already accepted/rejected

### Accept/Reject Buttons Not Working

**Problem**: Clicking buttons doesn't process the request

**Solutions**:
1. Ensure JavaScript is enabled
2. Check your internet connection
3. Refresh the page and try again
4. Try a different browser
5. Disable browser extensions temporarily
6. Contact support if issue persists

### Request Disappeared

**Problem**: A request was there but now it's gone

**Possible Reasons**:
1. The requester canceled the request
2. You accidentally accepted or rejected it
3. The requester's account was deactivated
4. System error occurred

**Solutions**:
1. Check your connections list (may have been accepted)
2. Check your notifications for any actions
3. Contact the requester directly if needed
4. Contact support if you believe it's an error

### Cannot Cancel Outgoing Request

**Problem**: No option to cancel a pending request

**Explanation**:
- Not all systems allow canceling requests
- This may be a system limitation
- The request will remain pending until the recipient responds

**Alternative**:
- Wait for the recipient to respond
- Contact the recipient directly if urgent
- Contact support for assistance

### Notification Badge Not Updating

**Problem**: Badge shows wrong number of requests

**Solutions**:
1. Refresh the page
2. Clear browser cache
3. Log out and log back in
4. Check if some requests are in a different status
5. Report the issue to support

## Privacy and Security

### What Others See

**When You Accept**:
- The requester is notified
- You appear in their connections list
- They can view your full profile
- They can message you

**When You Reject**:
- The requester may be notified (depending on settings)
- They cannot see your full profile
- They may not be able to send another request immediately
- No explanation is provided for the rejection

### Your Privacy

- **Request Privacy**: Only you can see your incoming and outgoing requests
- **Connection Privacy**: Your connections list is private
- **Profile Visibility**: Connected users can see your full profile
- **Messaging**: Only connections can message you
