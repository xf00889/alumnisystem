# Sending Requests

This guide explains how to find alumni and send connection requests to build your professional network.

## Overview

Connection requests allow you to reach out to fellow alumni and build your professional network. Once a connection request is accepted, you can message each other and see each other's full profile information.

## Who Can Send Connection Requests

- **Required**: Authenticated alumni account
- **Access Level**: All registered alumni users
- **Prerequisites**: 
  - Completed profile setup
  - Verified email address

## Finding Alumni to Connect With

### Method 1: Through Alumni Directory

#### Step 1: Access the Alumni Directory

1. Log in to your NORSU Alumni account
2. Click on **"Alumni Directory"** in the main navigation menu
3. The directory page will load with all registered alumni

**Expected Result**: You'll see the Alumni Directory page with alumni cards displayed in a grid layout.

![Alumni Directory](../../screenshots/alumni-directory-main.png)

#### Step 2: Browse or Search for Alumni

**Option A: Browse All Alumni**
- Scroll through the alumni cards
- Use pagination to view more alumni
- Look for familiar names or interesting profiles

**Option B: Use Search**
- Enter a name in the search bar at the top
- Click the search button or press Enter
- Results will filter to match your search

**Option C: Apply Filters**
- Use the filter options to narrow results by:
  - College
  - Campus
  - Graduation Year
  - Course/Program
  - Location
  - Employment Status

- Click **"Apply Filters"** to see filtered results

**Expected Result**: You'll see a refined list of alumni matching your criteria.

![Alumni Search and Filter](../../screenshots/alumni-search-filter.png)

For detailed information on searching and filtering, see [Alumni Directory - Search and Filtering](../alumni-directory/search-filtering.md).

### Method 2: Through Alumni Profiles

#### Step 1: View an Alumni Profile

1. Click on any alumni card in the directory
2. Or click **"View Profile"** button on an alumni card
3. The full profile page will open

**Expected Result**: You'll see the complete profile of the alumni, including their education, work experience, skills, and achievements.

![Alumni Profile Page](../../screenshots/alumni-profile-view.png)

#### Step 2: Review the Profile

Before sending a connection request, review:
- **Professional Background**: Work experience and current position
- **Education**: Academic history and qualifications
- **Skills**: Areas of expertise
- **Location**: Where they're currently based
- **Mutual Connections**: Alumni you both know (if displayed)

This helps you decide if you want to connect with this person.

## Sending a Connection Request

### From Alumni Directory

#### Step 1: Locate the Connect Button

On each alumni card, you'll see action buttons at the bottom:
- **"View Profile"** (blue button)
- **"Connect"** (teal button) - This is what you'll click

![Alumni Card with Connect Button](../../screenshots/alumni-card-connect.png)

#### Step 2: Click the Connect Button

1. Click the **"Connect"** button on the alumni card
2. The system will process your request

**Expected Result**: 
- A success message appears: "Connection request sent to [Alumni Name]"
- The button changes to **"Pending"** and becomes disabled
- The request is added to your outgoing requests list

![Connection Request Sent](../../screenshots/connection-request-sent.png)

### From Alumni Profile Page

#### Step 1: Open the Profile

1. Navigate to the alumni's profile page
2. Locate the connection action button near the top of the profile

#### Step 2: Send the Request

1. Click the **"Connect"** or **"Send Connection Request"** button
2. The system will process your request

**Expected Result**:
- A success message confirms the request was sent
- The button changes to show "Pending" status
- You're redirected back to the profile or directory

![Profile Connect Button](../../screenshots/profile-connect-button.png)

## Understanding Request Status

### Button States

The connection button displays different states:

1. **"Connect"** (Teal/Info Color)
   - No connection exists
   - You can send a request
   - Button is clickable

2. **"Pending"** (Gray/Disabled)
   - Request already sent
   - Waiting for response
   - Button is not clickable

3. **"Message"** (Blue/Primary Color)
   - Already connected
   - Opens messaging interface
   - Button is clickable

4. **"Accept Request"** (Green/Success Color)
   - They sent you a request
   - Click to accept
   - Button is clickable

![Connection Button States](../../screenshots/connection-button-states.png)

### Viewing Sent Requests

To see all your outgoing connection requests:

1. Click on **"Connections"** in the main navigation
2. Select **"Connection Requests"** or **"Requests"**
3. Click on the **"Outgoing Requests"** tab

**Expected Result**: You'll see a list of all pending requests you've sent, including:
- Recipient's name and photo
- Date request was sent
- Option to cancel the request (if available)

![Outgoing Requests List](../../screenshots/outgoing-requests.png)

## Connection Request Limitations

### What You Cannot Do

1. **Send to Yourself**
   - You cannot send a connection request to your own account
   - Error message: "You cannot send a connection request to yourself"

2. **Duplicate Requests**
   - You cannot send multiple requests to the same person
   - Error message: "Connection request already sent"

3. **Already Connected**
   - You cannot send a request to someone you're already connected with
   - The button will show "Message" instead

4. **Previously Rejected**
   - If your request was rejected, you may not be able to send another immediately
   - Error message: "Connection request was previously rejected"

5. **Blocked Connections**
   - If a connection is blocked, you cannot send requests
   - Error message: "Unable to send connection request"

### Request Limits

- **No Daily Limit**: You can send as many requests as needed
- **One Per User**: Only one pending request per user at a time
- **No Expiration**: Requests don't expire automatically

## After Sending a Request

### What Happens Next

1. **Notification Sent**: The recipient receives a notification about your request
2. **Email Alert**: They may receive an email notification (depending on their settings)
3. **Pending Status**: Your request appears in their incoming requests list
4. **Waiting Period**: You wait for them to accept or reject the request

### Possible Outcomes

1. **Accepted**: 
   - You receive a notification
   - You're now connected
   - You can message each other
   - Button changes to "Message"

2. **Rejected**:
   - You may receive a notification
   - Request is removed from pending
   - You may not be able to send another request immediately

3. **No Response**:
   - Request remains pending
   - No time limit for response
   - You can cancel if needed (feature may vary)

## Tips for Successful Connections

### Best Practices

1. **Complete Your Profile First**
   - Add a professional photo
   - Fill in your work experience
   - List your skills and achievements
   - A complete profile increases acceptance rates

2. **Connect with Relevant Alumni**
   - Classmates from your year
   - Alumni in your field or industry
   - People in your geographic area
   - Alumni with shared interests

3. **Be Strategic**
   - Don't send requests to everyone
   - Focus on building meaningful connections
   - Consider mutual connections

4. **Follow Up**
   - Once connected, send a message
   - Introduce yourself
   - Explain why you wanted to connect
   - Build the relationship

### Who to Connect With

**Good Candidates for Connection:**
- Former classmates and friends
- Alumni in your industry or field
- Professionals in companies you're interested in
- Alumni in your geographic area
- People with skills you want to learn
- Potential mentors
- Alumni with mutual connections

**Consider Carefully:**
- Alumni you don't know at all
- People outside your professional interests
- Alumni with incomplete profiles
- Users with no mutual connections

## Troubleshooting

### Connection Request Failed

**Problem**: Error message when trying to send request

**Solutions**:
1. Refresh the page and try again
2. Check if you're already connected
3. Verify the user's account is active
4. Check your internet connection
5. Log out and log back in
6. Clear browser cache

### Button Not Responding

**Problem**: Connect button doesn't work when clicked

**Solutions**:
1. Ensure JavaScript is enabled
2. Try a different browser
3. Disable browser extensions
4. Check for page loading errors
5. Refresh the page

### Cannot Find Alumni to Connect

**Problem**: Can't find specific alumni in directory

**Solutions**:
1. Check spelling of names
2. Try different search terms
3. Use filters to narrow results
4. Verify they have an account
5. Ask them for their username

### Request Not Showing as Sent

**Problem**: After clicking Connect, status doesn't change

**Solutions**:
1. Wait a few seconds for processing
2. Refresh the page
3. Check outgoing requests list
4. Try sending again if no error occurred
5. Contact support if issue persists

## Privacy and Security

### What Recipients See

When you send a connection request, the recipient sees:
- Your name and profile photo
- Your current position (if provided)
- Your graduation year and course
- Mutual connections (if any)
- Your profile summary

### What You Should Know

- **No Notifications to Others**: Only the recipient is notified
- **Private Action**: Other users don't see your pending requests
- **Reversible**: You may be able to cancel pending requests
- **No Penalty**: Rejected requests don't affect your account
