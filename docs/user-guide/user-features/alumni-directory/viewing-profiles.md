# Viewing Profiles

This guide explains how to view detailed alumni profiles, understand the different profile sections, and interact with profile information.

## Overview

Alumni profiles provide comprehensive information about fellow NORSU graduates, including their education history, work experience, skills, achievements, and contact information. Viewing profiles helps you learn more about alumni before connecting with them.

## Accessing Alumni Profiles

### From the Directory

1. Navigate to the Alumni Directory
2. Browse or search for an alumnus
3. Click the **"View Profile"** button on any alumni card
4. The full profile page will load

**Expected Result**: You'll see a detailed profile page with multiple sections of information.

![Alumni Profile Page](../../screenshots/alumni-profile-page.png)

### From Search Results

1. Use the search and filter features to find specific alumni
2. Click on any alumni card in the results
3. The profile page opens with complete information

### From Connections

1. Navigate to your Connections page
2. Click on any connection's name or profile picture
3. Their full profile opens

## Profile Page Layout

The profile page is divided into two main columns:

### Left Column (Main Content)
- Personal Information
- Professional Information
- Professional Experience
- Skills & Interests
- Additional Information

### Right Sidebar
- University Information
- Contact Options
- Documents

## Profile Header Section

### Profile Photo
- Displays at the top of the page
- Shows uploaded profile picture or default avatar
- 150x150 pixels, circular format
- Hover to see slight zoom effect

### Mentor Badge
- Green circular badge with teacher icon
- Appears in bottom-right corner of profile photo
- Only visible for verified mentors
- Indicates the alumnus is available for mentorship

### Name and Headline
- **Full Name**: Displayed prominently (large, bold text)
- **Job Title**: Current position and company (if provided)
- Example: "Software Engineer at Google"

### Quick Info Bar
- **Location**: City, province, and country
- **Email**: Primary email address
- **Graduation Year**: Displayed as "Class of [Year]"
- Each item has an icon for easy identification

![Profile Header](../../screenshots/alumni-profile-header.png)

## Personal Information Section

### What's Included

The Personal Information section displays:

#### Left Column
- **Full Name**: Complete name
- **Date of Birth**: Birth date (if provided)
- **Gender**: Male, Female, or Other

#### Right Column
- **Phone**: Contact phone number
- **Email**: Primary email address
- **Alternate Email**: Secondary email (if provided)
- **Address**: Complete address including city, province, and country

### Privacy Note
Some alumni may choose to hide certain personal information. If a field is not displayed, it means the alumnus hasn't provided that information or has set it to private.

![Personal Information](../../screenshots/alumni-personal-info.png)

## Professional Information Section

### Employment Details

This section shows current employment information:

#### Left Column
- **Employment Status**: Current work situation
  - Employed Full-Time
  - Employed Part-Time
  - Self-Employed
  - Unemployed
  - Further Studies
  - Retired
  - Internship/OJT
- **Job Title**: Current position
- **Current Company**: Employer name

#### Right Column
- **Industry**: Field of work
- **LinkedIn**: Link to LinkedIn profile (if provided)

### Understanding Employment Status

- **Employed Full-Time**: Working 40+ hours per week
- **Employed Part-Time**: Working less than 40 hours per week
- **Self-Employed**: Running own business or freelancing
- **Further Studies**: Pursuing additional education
- **Internship/OJT**: In training or internship program

![Professional Information](../../screenshots/alumni-professional-info.png)

## Professional Experience Section

### Timeline View

Professional experience is displayed in a vertical timeline format:

#### Timeline Features
- **Chronological Order**: Most recent position at the top
- **Timeline Markers**: Dots indicating each position
- **Current Position**: Green marker for current role
- **Date Ranges**: Start and end dates for each position

### Experience Entry Details

Each experience entry shows:

#### Position Information
- **Job Title**: Role or position name (large, bold)
- **Company Name**: Employer (blue text)
- **Date Range**: Start date to end date (or "Present" for current)
- **Location**: City and country (if provided)
- **Experience Type Badge**: Special designation (if applicable)
  - Career Path
  - Milestone
  - Achievement

#### Additional Details
- **Description**: Detailed role description
- **Key Achievements**: Notable accomplishments in the role
- **Responsibilities**: Main duties and tasks

### Current Position Indicator
- Current positions show a green timeline marker
- Badge displays "Present" instead of end date
- Appears at the top of the timeline

### Experience Types

The system tracks different types of experience:
- **Regular**: Standard work experience
- **Career Path**: Significant career milestones
- **First Job**: First professional position
- **Career Change**: Transition to new field

![Professional Experience Timeline](../../screenshots/alumni-experience-timeline.png)

Skills are shown as colored badges:
- **Blue badges**: Individual skills
- **Comma-separated**: Multiple skills listed
- **Clickable**: May be used for filtering (future feature)

**Example Skills**:
- Python
- JavaScript
- Project Management
- Data Analysis
- Leadership

Interests are shown as gray badges:
- **Gray badges**: Individual interests
- **Comma-separated**: Multiple interests listed

**Example Interests**:

- Travel

- Volunteering

### Empty State
If no skills or interests are provided:
- Message displays: "No skills or interests information available"
- Info icon indicates missing data

## Achievements Section

### Achievement Entries

The Achievements section may include:
- **Awards**: Professional or academic awards
- **Certifications**: Professional certifications
- **Publications**: Research papers or articles
- **Projects**: Notable projects completed
- **Recognition**: Special recognition received

### Achievement Details
Each achievement may show:
- **Title**: Achievement name
- **Type**: Category (Award, Certification, etc.)

- **URL**: Link to verification (if available)

## Additional Information Section

### Bio

  - Career goals
  - Professional philosophy
  - Personal interests
  - Motivations

### Achievements Text
- Free-form text about accomplishments

- Message displays: "No additional information available"

## Documents Section

### Document Statistics

At the top of the Documents section:
- **Total**: Total number of uploaded documents
- **Verified**: Number of verified documents
- **Pending**: Number awaiting verification

### Document List

Each document entry shows:

#### Document Information
- **Title**: Document name
- **Type**: Document category
  - Resume/CV
  - Certification
  - Diploma
  - Transcript of Records
  - Other
- **Upload Date**: When the document was uploaded

#### Verification Status
- **Verified Badge**: Green badge with checkmark
  - Indicates document has been verified by admin
- **Pending Badge**: Orange badge with clock icon
  - Indicates document awaits verification

#### Action Buttons
- **View Button**: Opens document in new tab
- **Download Button**: Downloads document to your device

### Document Types

Common document types include:
- **Resume/CV**: Current resume or curriculum vitae
- **Certification**: Professional certifications
- **Diploma**: Graduation diploma
- **Transcript of Records**: Academic transcript
- **Other**: Miscellaneous documents

### Privacy Note
Only documents the alumnus has chosen to share publicly are visible. Some documents may be restricted to connections only.

## Contact Options

### Available Contact Methods

Located in the right sidebar, contact options include:

#### Request Mentorship (For Mentors Only)
- **Icon**: Graduation cap
- **Text**: "Request Mentorship"
- **Action**: Opens mentorship request form
- **Availability**: Only for verified mentors

#### Connect / Message
- **Connect Button**: Send connection request
  - Blue button with user-plus icon
  - Text: "Connect" / "Send connection request"
- **Pending State**: Request already sent
  - Gray button with clock icon
  - Text: "Request Sent" / "Waiting for response"
  - Disabled (not clickable)
- **Message Button**: Send direct message
  - Teal button with chat icon
  - Text: "Message" / "Send direct message"
  - Available when already connected

#### Send Email
- **Icon**: Envelope
- **Text**: "Send Email"
- **Action**: Opens default email client
- **Shows**: Alumni's email address

#### View LinkedIn
- **Icon**: LinkedIn logo
- **Text**: "View LinkedIn"
- **Action**: Opens LinkedIn profile in new tab
- **Availability**: Only if LinkedIn profile is provided

### Connection States

The connection button changes based on your relationship:

1. **Not Connected**: Shows "Connect" button
2. **Request Sent**: Shows "Request Sent" (disabled)
3. **Request Received**: Shows "Accept" and "Reject" buttons
4. **Connected**: Shows "Message" and "Remove Connection" buttons

## Interacting with Profiles

### Sending Connection Requests

To connect with an alumnus:

1. Click the **"Connect"** button in the Contact section
2. A loading spinner appears briefly
3. Button changes to "Request Sent" (disabled)
4. Success message appears: "Connection request sent successfully"
5. Wait for the alumnus to accept your request

**Expected Result**: The button becomes disabled and shows "Request Sent" status.

### Accepting Connection Requests

If you receive a connection request:

1. Two buttons appear: "Accept" and "Reject"
2. Click **"Accept"** to connect
3. Button changes to "Message"
4. Success message appears
5. You can now message the alumnus

### Messaging Connected Alumni

To message a connected alumnus:

1. Ensure you're connected (button shows "Message")
2. Click the **"Message"** button
3. Messaging interface opens
4. Type and send your message

### Removing Connections

To remove a connection:

1. Click the **"Remove Connection"** button
2. Confirm the action in the popup
3. Connection is removed
4. Button changes back to "Connect"

### Sending Email

To send an email:

1. Click the **"Send Email"** button
2. Your default email client opens
3. Alumni's email is pre-filled in the "To" field
4. Compose and send your email

### Viewing Documents

To view or download documents:

1. Scroll to the Documents section
2. Find the document you want to view
3. Click **"View"** to open in browser
4. Or click **"Download"** to save to your device

**Note**: Some documents may require verification before viewing.

## Profile Completion Indicator

### For Your Own Profile

When viewing your own profile:
- Profile completion percentage may be displayed
- Shows which sections need updating
- Helps you maintain a complete profile

### For Other Profiles

When viewing other alumni profiles:
- Completion percentage is not shown
- You see only the information they've chosen to share

## Mobile Profile Viewing

### Mobile Layout

On mobile devices:
- Profile photo appears at the top
- Sections stack vertically
- Contact buttons are full-width
- Timeline is optimized for narrow screens

### Mobile Navigation

- **Scroll**: Swipe up/down to navigate sections
- **Tap**: Tap buttons to interact
- **Back**: Use browser back or "Back to Directory" button

### Mobile Tips

- **Portrait Mode**: Best viewing experience
- **Zoom**: Pinch to zoom on profile photo
- **Links**: Tap once to activate links
- **Documents**: Tap "View" to open in new tab

## Privacy and Visibility

### What You Can See

You can view:
- Information the alumnus has made public
- Documents shared publicly or with connections
- Contact information if provided

### What You Cannot See

You cannot view:
- Private or hidden information
- Restricted documents
- Information not provided by the alumnus

### Respecting Privacy

- Use contact information for professional purposes only
- Respect connection request responses
- Don't share profile information without permission
- Report inappropriate profiles to administrators

## Tips for Effective Profile Viewing

### Finding Relevant Information

1. **Scan the Header**: Get quick overview from name, title, and location
2. **Check Experience**: Review professional background
3. **Review Education**: Verify academic credentials
4. **Look for Common Ground**: Find shared interests or experiences
5. **Check Documents**: Verify qualifications if relevant

### Before Connecting

1. **Review Full Profile**: Understand their background
2. **Check Common Interests**: Find connection points
3. **Read Bio**: Understand their goals and interests
4. **Verify Credentials**: Check education and experience
5. **Prepare Message**: Plan what to say when connecting

### Networking Effectively

1. **Personalize Requests**: Mention specific details from their profile
2. **Find Common Ground**: Reference shared experiences
3. **Be Professional**: Maintain professional tone
4. **Respect Boundaries**: Don't be pushy if request is declined
5. **Follow Up**: Send a thank you message after connecting

## Troubleshooting

### Profile Not Loading

If a profile doesn't load:
- **Refresh**: Try refreshing the page
- **Check Connection**: Verify your internet connection
- **Clear Cache**: Clear browser cache and try again
- **Try Later**: The profile may be temporarily unavailable

### Missing Information

If expected information is missing:
- **Privacy Settings**: Alumni may have hidden certain information
- **Incomplete Profile**: Alumni may not have filled out all sections
- **Not Applicable**: Some fields may not apply to all alumni

### Buttons Not Working

If contact buttons don't work:
- **Check Login**: Ensure you're logged in
- **Refresh Page**: Try refreshing the browser
- **Check Status**: Verify your connection status
- **Try Different Browser**: Test in another browser

### Documents Not Opening

If documents won't open:
- **Check Format**: Ensure your device supports the file type
- **Download Instead**: Try downloading instead of viewing
- **Check Permissions**: You may not have access to the document
- **Contact Admin**: Report persistent issues
