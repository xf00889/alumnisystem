## Screenshot Guidelines

- Use a consistent browser and screen resolution
- Capture full-width screenshots when possible
- Highlight important UI elements with red boxes or arrows
- Use sample data that looks realistic but doesn't contain real personal information
- Ensure text is readable in screenshots
- Save screenshots in PNG format for best quality
- Use descriptive filenames

## Adding Screenshots to Documentation

To add a screenshot to the documentation, use the following markdown syntax:

```markdown
![Alt text description](../../screenshots/surveys/filename.png)
```

Example:
```markdown
![Surveys List Page](../../screenshots/surveys/surveys-list.png)
```
