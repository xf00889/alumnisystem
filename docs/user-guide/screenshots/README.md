## Screenshot Guidelines

### Naming Convention

Use descriptive, kebab-case names that clearly indicate what the screenshot shows:

- `login-page.png` - The login page
- `profile-edit-form.png` - The profile editing form
- `event-list-view.png` - The events list page
- `create-announcement-step-1.png` - First step of creating an announcement

### Image Requirements

- **Format**: PNG (preferred) or JPG
- **Resolution**: Minimum 1280x720 pixels for clarity
- **File Size**: Optimize images to keep under 500KB when possible
- **Content**: 
  - Crop to show only relevant UI elements
  - Remove or blur any sensitive personal information
  - Use sample/test data, not real user data
  - Ensure text is readable

### Annotations

When adding annotations to screenshots:
- Use red boxes or arrows to highlight important elements
- Add numbered callouts for step-by-step instructions
- Keep annotations clear and minimal
- Use a consistent style across all screenshots

### Updating Screenshots

When the UI changes:
1. Identify affected screenshots
2. Capture new screenshots following the guidelines
3. Replace old screenshots with the same filename
4. Update documentation if steps have changed

## Tools for Screenshots

Recommended tools for capturing and editing screenshots:
- **Windows**: Snipping Tool, Snip & Sketch, or ShareX
- **macOS**: Screenshot utility (Cmd+Shift+4)
- **Cross-platform**: Greenshot, Lightshot
- **Annotation**: Paint.NET, GIMP, or online tools like Photopea

## Placeholder Images

Until actual screenshots are captured, use placeholder images with descriptive text indicating what should be shown.
