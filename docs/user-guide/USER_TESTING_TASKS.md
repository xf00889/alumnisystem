# User Testing Task Checklists
## NORSU Alumni System Documentation

This document provides detailed task checklists for different user roles during documentation testing.

---

## Instructions for Testers

1. **Choose your role**: Select the checklist that matches your assigned role
2. **Follow the documentation**: Use the linked documentation to complete each task
3. **Take notes**: Document any issues, confusion, or suggestions
4. **Capture screenshots**: Take screenshots as you complete tasks
5. **Complete feedback form**: Fill out the feedback form for each task
6. **Be honest**: Your honest feedback helps us improve

**Important**: Try to complete tasks using ONLY the documentation. Don't rely on prior knowledge or ask for help unless absolutely stuck.

## General Testing Notes

### For All Testers

**Before You Start**:
- [ ] Review the testing plan
- [ ] Understand your role
- [ ] Have feedback form ready
- [ ] Set up screenshot capture tool
- [ ] Allocate sufficient time

**During Testing**:
- [ ] Follow documentation only (don't use prior knowledge)
- [ ] Take notes as you go
- [ ] Capture screenshots at each step
- [ ] Document any confusion immediately
- [ ] Note any errors or inaccuracies

**After Testing**:
- [ ] Complete feedback form
- [ ] Submit screenshots
- [ ] Review your notes
- [ ] Submit all materials
- [ ] Be available for follow-up questions

### Tips for Effective Testing

1. **Be Thorough**: Don't skip steps, even if they seem obvious
2. **Be Honest**: Negative feedback is valuable
3. **Be Specific**: "Confusing" is less helpful than "I didn't understand what 'primary education' means"
4. **Think Aloud**: If doing live testing, verbalize your thoughts
5. **Note Assumptions**: If you assumed something, note it
6. **Compare**: Think about how this compares to other documentation you've used
